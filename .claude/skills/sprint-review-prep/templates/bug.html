原因分析

🔍 フロントエンドとバックエンドのコード定義ミスマッチ

shopping_card.dart の _storeLabel() は購入場所の文字列キー（'supermarket' 等）でswitch文を書いていた。しかしバックエンドAPIが実際に返すのは数値コード（'1', '2', '3'）だったため、どのcaseにもマッチせず空文字 → IDがそのまま表示される状態になっていた。

購入場所 フロント側の期待値（誤） バックエンドAPIが返す値（実際）
スーパー 'supermarket' ≠ '1'
オンライン 'online' ≠ '2'
ドラッグストア 'drug_store' ≠ '3'

改修内容

lib/features/home/presentation/widgets/shopping_card.dart +3 / -3
129 String _storeLabel(String storeType) {
130 return switch (storeType) {
131- 'supermarket' => l10n.storeTypeSupermarket,
132- 'online' => l10n.storeTypeOnline,
133- 'drug_store' => l10n.storeTypeDrugStore,
131+ '1' => l10n.storeTypeSupermarket,
132+ '2' => l10n.storeTypeOnline,
133+ '3' => l10n.storeTypeDrugStore,
134 _ => storeType,
135 };
136 }
💡 AC3（フォールバック処理）は既存の _ => storeType がそのまま対応。未知のコードが来ても画面クラッシュせず、コードをそのまま表示する。
