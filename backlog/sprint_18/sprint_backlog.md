## Issue #43: ダークモードへ対応したい

**GitHub**: https://github.com/ryokkon624/hw-hub-manage/issues/43  
**ブランチ**: `feature/43-dark-mode`  
**コミット参照**: `(ryokkon624/hw-hub-manage#43)`

---

### 概要

#### 現状
サイドバーは暗い色だが、メインコンテンツは明るい固定色。

#### 問題点
夜間の利用時に眩しく、ユーザーの好みに合わせたカスタマイズ性がない。

#### 改善案
システム設定連動、かつ、手動で切り替え可能なフルダークモードを実装。
TailwindのdarkクラスをCSS変数と組み合わせ、全てのカラートークンを最適化する。

#### 設定値の保管場所
**「Local Storage と DB の併用」**がベストではあるが、当バックログでは最初の一歩としてLocal Storageのみで良い。

- Local Storage にもつ理由（必須）: 画面が表示される「直前」にテーマを判定するため。DBだけに保存していると、APIで設定を取得するまでの数ミリ秒〜数秒間、画面がライトモードで表示されてしまい、その後に急に黒くなる「チカチカ（Flash of unstyled content）」が発生する。Local Storageなら即座に読み込めるので、これを防げる。
- DB にもつ理由（推奨）: 「PCではダークモードにしたのに、スマホで見たらライトモードのまま」という事態を防ぐため。ログイン後にDBから設定を読み込み、Local Storageを上書きすることで、どのデバイスでも同じ体験を提供可能。
- 結論: 「基本はLocal Storageを参照し、ログイン後にDBの値で同期する」という流れが良い。

#### CSS変数を利用し効率的な実装をすること
実装例
```
:root {
  /* ライトモードの色 */
  --color-hwhub-surface: #f7f8f6;
  --color-hwhub-heading: #0f172a; /* slate-900 */
}

.dark {
  /* ダークモードの色 */
  --color-hwhub-surface: #1a2e1a; /* 深い緑がかった黒など */
  --color-hwhub-heading: #f8fafc; /* 白に近い色 */
}

/* クラスの定義は変数を参照するようにする */
.bg-hwhub-surface {
  background-color: var(--color-hwhub-surface);
}
```

#### 色を考える際のポイント
単に白を黒にするのではなく、以下のような配慮し「プレミアム感」を出すこと。
- 純粋な黒 (#000) を避ける: 少しグレーやネイビー、ダークグリーンを混ぜた黒の方が、目に優しく高級感がでる。
- コントラスト比: 文字が読みやすいよう、コントラストを確保しつつも眩しくない色を選ぶ。

## DEVへの追加指示

**色の定義はりょこさんが事前に決定済み。**  
`backlog/sprint_18/main.css` に以下が格納されている：

- ライトモード／ダークモード両方の CSS カスタムプロパティ（`:root` / `:root.dark`）
- OS連動用の `@media (prefers-color-scheme: dark)` ブロック
- `@layer utilities` 内の `.bg-hwhub-*` / `.text-hwhub-*` などのユーティリティクラス
- Tailwind v4 用の `@custom-variant dark` 定義

**実装時はこのファイルの内容を `src/assets/main.css` にそのまま取り込み、独自に色を考えなくてよい。**  
既存の `.vue` ファイルで使われているカラートークン（例: `bg-hwhub-surface` など）も、変数参照に置き換える形でそのまま使える。
