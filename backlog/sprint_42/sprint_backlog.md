## Issue #116 — 家事割り当て画面のスワイプモードで自分に割り当てるスワイプ方向が逆になっている

**GitHub Issue**: https://github.com/ryokkon624/hw-hub-manage/issues/116
**ラベル**: bug
**ブランチ**: `fix/116-118-mobile-assign-screen-bugs`

### 発生事象

家事割り当て画面のスワイプモード（Tinder風カード）で、自分に割り当てるスワイプ方向が通常リストモードと逆になっている。通常リストモードでは左スワイプが「自分にする」だが、スワイプモードでは逆の方向になっている。

### ユーザーストーリー

**As a** HwHubモバイルアプリのユーザー
**I want to** 通常リストモードとスワイプモードで同じ方向のスワイプで自分への割り当てができるようにしたい
**So that** 画面内で操作方向が統一されて直感的に操作できる

### Acceptance Criteria

- [x] AC1: スワイプモードで左スワイプすると「自分に割り当てる」アクションが実行される（通常リストモードと同じ方向に統一）
- [x] AC2: スワイプモードで右スワイプすると「次へ（スキップ）」アクションが実行される
- [x] AC3: 「他メンバーのタスクを奪う」スワイプモードでも左スワイプで割り当て・右スワイプでスキップの方向に統一される

### 備考

- 対象画面: 家事割り当て画面スワイプモード（Tinder風カード画面）
- Sprint 41 Sprint Reviewで発覚

## Issue #117 — 家事割り当て画面のスワイプモードのOverdueカードがレイアウトオーバーフローしている

**GitHub Issue**: https://github.com/ryokkon624/hw-hub-manage/issues/117
**ラベル**: bug
**ブランチ**: `fix/116-118-mobile-assign-screen-bugs`

### 発生事象

「自分に割り当てる」「他メンバーのタスクを奪う」スワイプモードにおいて、Overdueのカードで「実施日」と「現在の担当」の間に「BOTTOM OVERFLOWED BY 23 PIXELS」と表示されている。

### ユーザーストーリー

**As a** HwHubモバイルアプリのユーザー
**I want to** スワイプモードのカードでOverdueタスクが正しく表示されてほしい
**So that** 過去の未完了タスクも崩れなくスワイプ操作できる

### Acceptance Criteria

- [x] AC1: 「自分に割り当てる」「他メンバーのタスクを奪う」両スワイプモードで、OverdueカードのBOTTOM OVERFLOWEDエラーが表示されない
- [x] AC2: Overdueカードの実施日はMyTasks画面の「過去の家事」カードと同様に赤字・背景色が薄い赤で表示される（「自分に割り当てる」「他メンバーのタスクを奪う」の両スワイプモードで対応）

### 備考

- 対象画面: 家事割り当て画面スワイプモード（Tinder風カード画面）
- 参照: MyTasks画面の「過去の家事」カードのOverdue表現

## Issue #118 — 家事割り当て画面のユーザアイコンに画像が表示されない

**GitHub Issue**: https://github.com/ryokkon624/hw-hub-manage/issues/118
**ラベル**: bug

### 備考

- 対象画面: 家事割り当て画面（メンバーサマリ・家事カード担当者欄）
- Sprint 41 Sprint Reviewで発覚
