原因分析

🔍 各画面が独自にエラーWidgetを実装しており、共通基盤がなかった

home_page.dart の _ErrorBody はプライベートクラスとして定義されており、他画面から参照できない状態だった。そのため各画面が独自にエラー表示を実装した結果、デザイン・ボタン種別（FilledButton / TextButton / ElevatedButton）・エラーメッセージ変換ロジックがすべて異なる状態となっていた。

また、エラーメッセージの変換ロジック（例外種別 → i18n キー）も home_page.dart / my_tasks_page.dart / shopping_list_page.dart の3箇所にそれぞれベタ書きされており、家事割り当て画面と家事設定画面は e.toString() をそのまま表示する（内部例外クラス名がユーザーに見える）状態だった。

問題の核心

共通 Widget がなく、各画面がそれぞれ独自実装したため、画面ごとに見た目・ボタン・メッセージ変換の3点すべてがバラバラになっていた。

改修前
home_page.dart: private _ErrorBody（他から参照不可）
my_tasks_page.dart: private _ErrorBody（別デザイン）
shopping_list_page.dart: private _ErrorBody（ElevatedButton）
housework_assign: Center(child: Text(e.toString()))
housework_create: Center(child: Text(e.toString()))
→
改修後
core/ui/app_error_view.dart: 共通 AppErrorView Widget
resolveErrorMessage() 変換関数も同ファイルで共有
全5画面が AppErrorView を参照（同一デザイン）
!マーク + メッセージ + FilledButton に統一

動作確認ポイント（ユーザー確認用）

  • 1
    家事割り当て画面でエラー発生時に !マーク + メッセージ + 再読み込みボタンが表示されること 機内モードや開発サーバー停止でネットワークエラーを発生させて確認。ホーム画面と同じ見た目になっているか確認
  • 2
    My Tasks画面でエラー発生時に同一デザインが表示されること エラー画面の「再読み込み」ボタンをタップするとリロードされること（ボタンが TextButton ではなく FilledButton になっていること）
  • 3
    買い物リスト画面でエラー発生時に同一デザインが表示されること ボタンが ElevatedButton ではなく FilledButton に変わっていること
  • 4
    家事設定（新規作成）画面でエラー発生時に !マーク + メッセージ + 再読み込みボタンが表示されること 以前は内部例外クラス名（例: NetworkException: Failed host lookup）がそのまま表示されていたが、「ネットワークエラーが発生しました。」等のユーザー向けメッセージになっていることを確認
  • 5
    再読み込みボタンをタップするとデータが再取得されること いずれの画面でも、ボタンタップ → 通信成功 → 正常な画面に戻れることを確認（ネットワーク復旧後）

再発防止

AppErrorView を共通 Widget として core/ui/ に配置したことで、今後の新規画面では自然とこのWidgetを使う設計になる。また各要素に Key を付与したことでウィジェットテストが容易になっている。
