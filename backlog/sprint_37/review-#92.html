Sprint 37 Review

#92 — 購入済みタブの件数バッジを非表示にし、
Web SP版と統一する

Sprint 37 #92 feature mobile

なぜこの改修が必要だったか

問題

購入済みタブに件数バッジが表示されていたが、Web SP版では購入済みタブに件数バッジを表示していない。「購入済みは最終到着地なので何件あっても重要ではない」というUX上の考え方から、Web版と統一するために非表示にすることにした。

改修前（購入済みにもバッジ）
未購入 5
かご 3
購入済み 12
改修後（購入済みはバッジなし）
未購入 5
かご 3
購入済み

AC別 実装内容

✓
AC1
購入済みタブのタブラベル横に件数バッジが表示されない

_TabItem に showCount: bool = true パラメータを追加し、件数バッジの表示条件を count > 0 から showCount && count > 0 に変更した。ShoppingTabBar から購入済みタブのみ showCount: false を渡す。

50 ),
...

💡 デフォルト値を true にしているため、既存の未購入・かごタブは showCount を指定しなくても件数バッジが表示され続ける。既存コードへの影響ゼロ。
✓
AC2
未購入・かごタブの件数バッジは引き続き表示される（リグレッションなし）

未購入・かごタブの _TabItem には showCount: true を明示的に渡した（デフォルト値と同じだが意図を明示する目的）。テストで未購入・かごタブのバッジ表示を確認している。

動作確認ポイント（りょこさん確認用）

  • 1
    購入済みタブのラベル横に件数バッジが表示されないこと 何件の購入済みアイテムがあっても、タブに数字が出ないこと
  • 2
    未購入タブのラベル横に件数バッジが引き続き表示されること 未購入アイテムが1件以上あればバッジで件数が表示されること
  • 3
    かごタブのラベル横に件数バッジが引き続き表示されること かごに1件以上あればバッジで件数が表示されること
  • 4
    アクティブなタブのバッジ色が緑、非アクティブは灰色になっていること 未購入・かごタブのバッジ色がタブのアクティブ状態に連動していること
