原因分析

🔍 setFromFavorite / setFromHistory で memo フィールドが copyWith に含まれていなかった

shopping_item_new_notifier.dart の setFromFavorite / setFromHistory は、copyWith で name / storeType / sourceShoppingItemId のみをセットしていた。memo フィールドが含まれていないため、お気に入り・購入履歴に紐づくメモが一切反映されず、メモ欄は前の入力内容のまま残っていた。

フィールド 修正前の setFromFavorite 修正後
name copyWith に含む → 変更なし
storeType copyWith に含む → 変更なし
sourceShoppingItemId copyWith に含む → 変更なし
memo copyWith に含まない（漏れ） + memo: item.memo ?? '' を追加

💡 メモがない場合は memo ?? '' で空文字をセットし、以前の入力内容を確実に消す。ShoppingItemDto / ShoppingItemHistorySuggestionDto 両方に memo フィールドが存在することを確認済み。

AC確認

✓
AC1
お気に入りからメモなしのアイテムを選択すると、メモ欄が空欄に更新される
memo: item.memo ?? '' により、メモが null の場合は空文字がセットされ、メモ欄が空欄になる。
✓
AC2
お気に入りからメモありのアイテムを選択すると、メモ欄が選択アイテムのメモで更新される
memo: item.memo ?? '' により、メモが存在する場合はそのままメモ欄に反映される。
✓
AC3
購入履歴からの選択時も同様にメモ欄が正しく更新される
setFromHistory にも同様の修正を適用。memo: suggestion.memo ?? '' でメモなしは空欄、メモありは選択履歴のメモで更新される。
✓
AC4
他の入力欄（アイテム名・購入場所等）の更新動作はリグレッションしない
name / storeType / sourceShoppingItemId の copyWith 引数はそのまま維持。変更はメモフィールドの追加のみ。
