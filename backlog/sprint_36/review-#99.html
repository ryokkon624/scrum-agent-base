原因分析

🔍 catch ブロックが rethrow のみで、ユーザー通知まで実装されていなかった

AppSnackBar（core/ui/app_snack_bar.dart）・AppException（core/network/app_exception.dart）の共通エラー通知インフラは実装済みだった。しかし catch (_) {} の握りつぶし修正時に、これらのインフラの存在を把握せず rethrow で対応したため、ユーザー通知までの実装が未完了となっていた。

Notifier 末端での rethrow は Riverpod に吸収されるだけで UI には反映されない。そのため API エラーが発生しても画面に何も表示されず、操作が無言で失敗する状態だった。

改修内容

対象 Notifier 全てに同じパターンを適用した。shopping_item_detail_notifier.dart のリファレンス実装を全 feature に横展開。
