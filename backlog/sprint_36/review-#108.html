原因分析

🔍 IndexedStack 配下の一覧 Notifier が削除後 invalidate されていなかった

買い物リスト一覧画面は StatefulShellRoute.indexedStack 配下に配置されているため、AutoDispose が破棄されず古い state が保持される。

shopping_item_detail_page.dart は Sprint 35（#93）対応で toggleFavorite 後に ref.invalidate(shoppingListNotifierProvider) を呼んでいたが、deleteItem 完了（isDeleted 検知で context.pop() する箇所）では invalidate が漏れていた。そのため削除済みアイテムのカードが一覧に残り続けていた。

改修内容

lib/features/shopping/presentation/shopping_item_detail/shopping_item_detail_page.dart +2 / -0
29 // 削除後に前の画面に戻る
30 ref.listen(shoppingItemDetailNotifierProvider(itemId), (prev, next) {
31 if (next.isDeleted && !(prev?.isDeleted ?? false)) {
32 ScaffoldMessenger.of(context).showSnackBar(
33 SnackBar(content: Text(l10n.shoppingDetailToastDeleteSuccess)),
34 );
35+ // #108: 削除後に一覧を即時反映するためinvalidate
36+ ref.invalidate(shoppingListNotifierProvider);
37 context.pop();
38 }
💡 context.pop() の前に ref.invalidate を呼ぶことで、一覧画面に戻った瞬間に再フェッチが走り、削除後の最新状態が即時表示される。Sprint 35（#93）の toggleFavorite 後の invalidate と同じパターン。
