一覧へ戻ると該当カードがタブ移動されていない

Sprint 36 #107 bug mobile

詳細で「牛乳」を「かご」に変更して一覧へ戻る

買い物リスト（ステータス変更後に戻る）

シャンプー

原因分析

🔍 StatusStepSelector の onChanged が updateStatus 後に一覧 Notifier を invalidate していなかった

StatusStepSelector の onChanged は widget.notifier.updateStatus を直接渡していた。updateStatus で即時 API コールが走りサーバー側のステータスは変わるが、IndexedStack 配下で保持される一覧 Notifier の state は invalidate されないため古いまま残っていた。

一覧に戻ると IndexedStack が古い state を表示し続けるため、カードが変更前のタブに残ったまま見えていた。

💡 updateStatus の完了を await してから ref.invalidate を呼ぶことで、API の更新完了後に一覧の再フェッチが走る。Sprint 35（#93）の toggleFavorite 後の invalidate と同じパターン。

AC確認

✓
AC1
詳細画面でステータスを変更後（保存ボタン押下なし）、一覧に戻ると該当カードが変更後のタブに移動している
onChanged ラムダ内で await updateStatus(status) 後に ref.invalidate(shoppingListNotifierProvider) を呼ぶことで、一覧の再フェッチが即時実行される。
✓
AC2
未購入→かご・かご→購入済みいずれのステータス変更でも一覧に即時反映される
updateStatus の引数は ShoppingItemStatus enum 値で、どのステータス変更でも同じ onChanged ラムダが呼ばれるため両方のパターンに対応済み。
✓
AC3
保存ボタン押下で一覧へ戻る動作が引き続き正常に動作する（リグレッションなし）
保存ボタンの _save メソッドは変更対象外。onChanged ラムダの変更は StatusStepSelector のタップ時のみ影響するため、保存ボタン経由の動作はリグレッションなし。
