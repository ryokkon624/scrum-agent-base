原因分析

🔍 Dismissible の child Container に width 指定が漏れていた

swipeable_task_card.dart の Dismissible の child である Container に width の指定がなかったため、Flutter のデフォルト動作でコンテンツ（タスク名テキスト）の intrinsic な幅になっていた。短いタスク名なら小さなカード、長いタスク名なら大きなカードという状態になっていた。

Sprint 31（#80）でセクションコンテナ化と SwipeableTaskCard の margin 削除を行ったが、 その際に width: double.infinity の設定が漏れたことが直接原因。

  • Sprint 31 #80 カードレイアウトをweb SP版に合わせる対応で margin を削除。このとき width: double.infinity を追加し忘れた
  • Sprint 32 #82 Sprint 31 Review 指摘を受けて修正。Container に width: double.infinity を1行追加

💡 Flutter の Container はデフォルトでコンテンツサイズになる。width: double.infinity を指定することで、 親ウィジェット（セクションコンテナ）が提供する最大幅いっぱいに広がる。 セクション側（past_tasks_section.dart / future_tasks_section.dart）の変更は不要。
