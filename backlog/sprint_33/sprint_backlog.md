## #85: [mobile] 買い物リスト画面を実装する (#13)

**GitHub Issue**: https://github.com/ryokkon624/hw-hub-manage/issues/85
**ラベル**: feature
**ブランチ名**: `feature/85-mobile-shopping-list`

### ユーザーストーリー

**As a** HwHubモバイルアプリのユーザー
**I want to** 買い物アイテムをタブ切り替えとスワイプ操作で管理したい
**So that** 買い物中にかごへ入れる・購入済みにするを素早くこなせる

### Acceptance Criteria

- [x] AC1: 3タブ（未購入 / かご / 購入済み）を表示する。デフォルトは「未購入」タブ。各タブヘッダーに件数バッジを表示する
- [x] AC2: 未購入タブ - 購入場所フィルタ（世帯に登録されている場所を動的表示）でフロント側絞り込みができる
- [x] AC3: 未購入タブ - アイテムカードに購入場所カラーバー・品名・メモ（任意）・お気に入りアイコン・画像アイコンを表示する
- [x] AC4: 未購入タブ - 右スワイプ（emerald-500背景・「かごへ」ラベル）でステータスをかごに更新する（`PATCH /api/shopping-items/{id}/status`）
- [x] AC5: 未購入タブ - 左スワイプ（rose-500背景・「削除」ラベル）で確認ダイアログを表示し、確認後にアイテムを削除する（`DELETE /api/shopping-items/{id}`）
- [x] AC6: かごタブ - ヘッダーの「購入済みにする」ボタンで確認ダイアログ後に一括購入済み更新ができる（`PATCH /api/shopping-items/bulk-status`）
- [x] AC7: かごタブ - 右スワイプ（emerald-500背景・「購入済みに」ラベル）でステータスを購入済みに更新する
- [x] AC8: かごタブ - 左スワイプ（slate-400背景・「戻す」ラベル）でステータスを未購入に戻す
- [x] AC9: 購入済みタブ - 直近7日間の購入済みアイテムを日付グループで表示する。スワイプ操作なし
- [x] AC10: お気に入りアイコンタップでお気に入りをトグルできる（`PATCH /api/shopping-items/{id}/favorite`）
- [x] AC11: スワイプ操作はカードがドラッグに追従し、カード幅30%超えでアクション確定。背景に色とラベルを表示し、確定後にトーストを表示する
- [x] AC12: 「+ アイテムを追加」ボタンタップで#14買い物アイテム作成画面へ遷移できる
- [x] AC13: アイテムカードタップで#15買い物アイテム詳細画面へ遷移できる
- [x] AC14: 複数世帯所属時は世帯インジケーターバーを表示し、タップで世帯切り替えができる（HouseholdSwitcherはホームで実装済みのUIを再利用）
- [x] AC15: 画面表示時・世帯切り替え時にAPIからデータを取得し、ローディング中はインジケーターを表示する
- [x] AC16: テストカバレッジ ≥95%（Notifier・ウィジェットテスト）→ 98.7%達成

### 備考

- 優先順位の根拠：My Tasks完了後の次の主要機能。買い物時に最も使う頻度が高い画面
- 依存関係：HouseholdSwitcherはホーム（#66）で実装済みのUIを再利用
