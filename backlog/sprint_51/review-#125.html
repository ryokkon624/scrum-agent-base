原因分析

🔍 loginNotifierProvider に autoDispose が設定されていなかった

ログイン成功時は authNotifierProvider の state 変更 → router redirect で別画面に遷移するが、loginNotifierProvider は NotifierProvider（autoDispose 未設定）で実装されていたため、画面を離れてもメモリに残り続ける。

1
ログインボタン押下 → submit() が isLoading: true をセット
2
ログイン成功 → auth state 変更 → router が別画面にリダイレクト。submit() は isLoading を false に戻していなかった（修正前）
3
ログアウト → ログイン画面に戻る。loginNotifierProvider が破棄されていないため isLoading: true の古い state がそのまま残る
4
ログイン画面のボタンはスピナー表示 + canSubmit = false でタップ不可（永久ループ）

💡 根本対策: autoDispose を付けることで画面を離れた際にプロバイダーが破棄され、ログイン画面に戻るたびに初期状態から始まる。防御策として saveTokens() 後にも isLoading: false をセットし、予期しない例外でもスピナーが残らないよう catch (_) ブロックを追加。
