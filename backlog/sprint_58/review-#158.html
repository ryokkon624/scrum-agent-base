原因分析

🔍 logout() が HouseholdNotifier と選択世帯ID をリセットしていなかった

HouseholdNotifier は AsyncNotifier（AutoDispose なし）のグローバル Provider であるため、logout() を呼んでもキャッシュが自動破棄されない。 logout() は tokenStorage.clearTokens() と state = AuthUnauthenticated() のみ実行しており、HouseholdNotifier の invalidate も SharedPreferences.selectedHouseholdId のクリアも行っていなかった。

1
ユーザーAでログイン → HouseholdNotifier.build() が世帯リストを取得、SharedPreferences.selectedHouseholdId にユーザーAの世帯IDを保存
2
logout() は clearTokens() と state = AuthUnauthenticated() のみ。HouseholdNotifier invalidate なし・selectedHouseholdId クリアなし
3
ユーザーBでログイン → HouseholdNotifier はキャッシュされたまま build() が再実行されない → 前ユーザーAの世帯リストと selectedHousehold.id が state に残り続ける
4
homeNotifierProvider / houseworkAssignNotifierProvider 等が householdNotifierProvider.future を watch → 前ユーザーAの世帯IDでAPIを呼ぶ → ユーザーBはそのメンバーでないため 403
5
アカウント設定・おうち設定・通知・問い合わせは /api/users/me/... 経由で自身のデータを参照するため正常（発生事象の観察と一致）
Providerの種類 AutoDispose logout() 前の状態 修正前: logout() 後
householdNotifierProvider なし（グローバル） ユーザーAの世帯リスト そのまま残る
homeNotifierProvider なし ユーザーAの世帯IDでキャッシュ そのまま残る
houseworkAssignNotifierProvider なし ユーザーAの世帯IDでキャッシュ そのまま残る
accountSettingsNotifierProvider あり — 画面離脱時に自動破棄

💡 3つの操作を追加: (1) selectedHouseholdId を SharedPreferences から削除して世帯IDのキャッシュをクリア、(2) グローバルの householdNotifierProvider を invalidate して世帯リストをリセット、(3) IndexedStack に保持される homeNotifierProvider・houseworkAssignNotifierProvider を invalidate して各画面のキャッシュをクリア。 shoppingListNotifierProvider 等は AutoDispose のため画面離脱時に自動破棄されるため含めていない。

実装メモ（仕様外の判断・変更・妥協点）

invalidate 対象の選択: logout() で invalidate する Provider として homeNotifierProvider・houseworkAssignNotifierProvider（IndexedStack でキャッシュが残りやすい autoDispose なし Provider）と householdNotifierProvider（グローバル非 autoDispose Provider）を選択した。shoppingListNotifierProvider なども候補だが AutoDispose のため画面離脱時に自動破棄されるため含めなかった。
auth_notifier_test.dart への StorageKeys import 追加: logout() の selectedHouseholdId 削除テストに SharedPreferences の初期値設定と値確認が必要なため、storage_keys.dart を import した。
