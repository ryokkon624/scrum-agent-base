問題

買い物アイテムには householdId が必須キーとして存在する。世帯に所属していない状態でも「アイテムを追加」ボタンが活性のままだったため、追加操作を行うと householdId が取得できずエラーになる不正な状態になっていた（Sprint 46 Review 指摘 ④）。

AC別 実装内容

✓
AC1
世帯に所属していない状態では、買い物リスト画面の「アイテムを追加」ボタンが非活性（disabled）になる

_ShoppingBody で householdNotifierProvider を ref.watch し、selectedHousehold == null のとき ElevatedButton.onPressed: null（= disabled）にした。Flutter の ElevatedButton は onPressed が null のとき自動的にグレーアウトされてタップ不可になる。

AC2
世帯に所属している状態では、「アイテムを追加」ボタンが活性になる（既存動作と同じ）

householdAsync.valueOrNull?.selectedHousehold != null（= hasHousehold = true）のとき、従来通り onPressed: () => context.push(AppRoutes.shoppingNew) が渡される。ウィジェットテストで「世帯所属時は onPressed が非null」「世帯未所属時は onPressed が null」の両方を検証済み。

実装メモ（仕様外の判断・変更・妥協点）

• household ローディング中も一時的に非活性になる: householdAsync.valueOrNull == null には「ローディング中」と「world帯未所属」の両方が含まれる。_ShoppingBody が表示される（= shopping が AsyncData）タイミングでは household も概ね解決済みのため実害なしと判断した。ローディング状態を分けた厳密な判定（householdAsync.isLoading / householdAsync.hasValue の組み合わせ）は現状では過剰と判断し採用しなかった。

動作確認ポイント（りょこさん確認用）

  • 1
    世帯に所属していないアカウントで買い物リスト画面を開き、「アイテムを追加」ボタンがグレーアウトしてタップできないこと 世帯未所属のテストアカウントでログイン → 買い物タブ → ボタンがグレー色で表示され、タップしても何も起きないことを確認
  • 2
    世帯に所属しているアカウントでは「アイテムを追加」ボタンが緑色で活性になっており、タップすると追加画面に遷移すること 通常のテストアカウント（世帯所属済み）でログイン → 買い物タブ → ボタンが緑色で表示され、タップすると買い物アイテム追加画面に進めることを確認
  • 3
    ボタンの文字「+ アイテムを追加」は両状態で表示されていること ボタン自体は活性/非活性どちらの状態でも画面上に表示され続けること（非活性時に消えたりしないこと）
