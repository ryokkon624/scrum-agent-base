#136 — [mobile] 家事設定一覧画面を実装する

Sprint 47 #136 feature mobile

なぜこの実装が必要だったか

背景

世帯の家事マスタ（家事の名称・周期・担当者設定）をモバイルアプリで管理できる画面が存在しなかった。 設定メニューから家事マスタの閲覧・新規作成・編集へアクセスできる起点となる一覧画面が必要だった。

実装前
設定メニューに「家事設定」項目はあるが画面未実装（プレースホルダー）
家事マスタの一覧・カテゴリフィルタ・ページネーション機能なし
新規作成・編集画面への遷移経路がない
→
実装後
HouseworkListPage（/settings/housework）が設定メニューから遷移可能に
カテゴリドロップダウンフィルタ + 10件/ページのページネーション
「家事を追加する」ボタンで新規作成画面へ、カードタップで編集画面へ遷移
世帯切り替え時に自動再取得（ref.watch(householdNotifierProvider.future)）

AC4
ページネーションが10件/ページで動作し、フィルタ後の件数で再計算される

HouseworkListState に pagedHouseworks / totalPages getter を持ち、filteredHouseworks を元に計算する。カテゴリ切り替え時に currentPage=1 にリセットされるため、フィルタ後件数で自動的に再計算される。

lib/features/housework_settings/presentation/housework_list/housework_list_state.dart getter追加
37+ /// 現在ページの家事一覧（ページネーション適用済み）
38+ List<HouseworkDto> get pagedHouseworks {
39+ final filtered = filteredHouseworks;
40+ final start = (currentPage - 1) * _pageSize; // _pageSize = 10
41+ if (start >= filtered.length) return [];
42+ final end = (start + _pageSize).clamp(0, filtered.length);
43+ return filtered.sublist(start, end);
44+ }
45+
46+ /// 総ページ数
47+ int get totalPages {
48+ final count = filteredHouseworks.length;
49+ if (count == 0) return 1;
50+ return ((count - 1) ~/ _pageSize) + 1;
51+ }

AC7
世帯切り替え時に一覧が再取得される

AsyncNotifier.build() 内で ref.watch(householdNotifierProvider.future) を await することで、世帯切り替え時に build() が自動的に再実行される。AutoDispose + Riverpod の再計算メカニズムを活用しており、追加実装は不要。

💡 ref.watch(householdNotifierProvider.future) により、householdNotifier が更新されると Notifier 全体が再 build されて新しい世帯の一覧が取得される。
