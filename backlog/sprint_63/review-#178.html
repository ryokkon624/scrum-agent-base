Sprint 63 Review

#178 — 問い合わせ起票時のUIクライアント・バージョン情報を
t_inquiryに保存する

Sprint 63 #178 feature DB Backend Frontend Mobile

なぜこの改修が必要だったか

課題

問い合わせがWebブラウザとモバイルアプリのどちらから届いたか、またどのバージョンを使っていたかを把握できず、バージョン固有の不具合の早期特定が困難だった。問い合わせ対応時に「Webですか？アプリですか？」と利用者に追加確認する手間が発生していた。

改修前
t_inquiry にクライアント情報カラムなし
問い合わせ詳細に利用環境の情報が表示されない
バージョン固有の不具合か確認するには利用者への追加質問が必要
Web/Mobileどちらからの問い合わせか判別不可
→
改修後
起票時のクライアント種別（web/mobile）を自動記録
UIバージョン・APIバージョンも同時に保存
問い合わせ詳細画面でクライアント情報を即座に確認できる
利用者の入力フォームへの追加負担はゼロ（自動付与）
Web/Mobile → 問い合わせ送信時に uiClient / uiVersion / apiVersion を自動付与してAPIコール
Backend → リクエストを検証し、InquiryModel 経由で t_inquiry に保存
DB → ui_client / ui_version / api_version を NOT NULL で保持
Backend → 詳細取得APIのレスポンスに3フィールドを追加
Web/Mobile → 問い合わせ詳細画面のカテゴリバッジ上部にクライアント情報を表示
