全員が2段に折り返して表示される

原因分析

🔍 水平スクロールListViewにより画面幅を超えたメンバーが不可視になっていた

member_summary_strip.dart が SizedBox(height: 72) の中に ListView(scrollDirection: Axis.horizontal) を使っていた。この構造では横幅に収まらないチップは右にスクロールしなければ見えず、ユーザーが水平スクロールできることに気づかない場合、1名以上が欠けているように見える。

修正前（水平スクロール）
SizedBox(height: 72,
  child: ListView(
    scrollDirection: Axis.horizontal,
    ...
  )
)
修正後（Wrap折り返し）
Padding(
  padding: ...,
  child: Wrap(
    spacing: 8,
    runSpacing: 8,
    ...
  )
)

Web版SPモードはメンバーチップが画面幅に収まらない場合に2段で折り返して表示するデザインになっている。モバイルも同様のUIにすることで一貫性を保つ。

💡 Wrap ウィジェットは子ウィジェットが親の横幅に収まらなくなると自動的に次の行に折り返す。spacing でチップ間の水平間隔、runSpacing で行間の垂直間隔を指定している。固定高さ（SizedBox(height: 72)）が不要になり、メンバー数に応じて高さが自動調整される。

AC確認

✓
AC1
メンバーサマリにおうちの全メンバーが表示される
Wrap に置き換えることで、全メンバーのチップが画面内に折り返して表示される。水平スクロールで隠れるメンバーがいなくなった。
✓
AC2
横幅に収まらない場合はWeb版SPモード同様に2段で表示される
Wrap(spacing: 8, runSpacing: 8) により、チップが横幅を超えると自動的に次の段に折り返す。Web版SPモードの2段表示と同じ動作になっている。
