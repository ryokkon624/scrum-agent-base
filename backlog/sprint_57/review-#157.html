原因分析

🔍 go_router 14.x の Navigator ツリー競合による Hero キー重複

/notifications（通知センター）は StatefulShellRoute.indexedStack のシェル外の独立ルートとして定義されている。そこから NotificationLinkNavigator.navigate() 内で context.push(AppRoutes.tasks) 等のシェル内ルートへ push すると、以下の状況が発生する：

Navigator ツリーの状態（バグあり）
シェル外 Navigator:
/notifications
+
/tasks（シェル付き）← ここが問題
シェル内 Navigator:
/tasks
→ 両方の Navigator に暗黙 Hero（AppBar 戻る矢印）が同時に存在し、
　 HeroController キーが重複 → AssertionError

なお、ポップオーバー（notification_popover.dart）経由の遷移は既に Navigator.of(context).pop() でポップオーバーを閉じてから navigate() を呼んでいるため、同問題は発生しない。通知センターページ（notification_center_page.dart）経由の遷移のみが問題。

修正後の遷移フロー
1. 通知センターで通知をタップ
2. navigate() 内で現在のルートが /notifications と判定
3. context.pop() → 通知センターが閉じる（シェル外 Navigator がクリア）
4. context.push(AppRoutes.tasks) → マイタスク等に遷移（Hero 重複なし）

実装メモ（仕様外の判断・変更・妥協点）

context.canPop() チェックの追加

context.pop() の前に canPop() でスタックに戻れるルートがあるかチェックを追加した。テスト環境で initialLocation が /notifications の場合はポップできないため、チェックなしだと GoError が発生する。実際のアプリでは通知センターは他のルートからpushされるため canPop() は常に true になるが、防御的実装として残した。
