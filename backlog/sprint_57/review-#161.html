問題

おうちの様子カードの積み上げ棒グラフはタッチが無効（BarTouchData(enabled: false)）になっており、各メンバーの日別タスク件数を確認するには凡例の色から目視で読み取る必要があった。Web版（HouseholdSituationCard.vue）では Chart.js のツールチップが既に有効になっており、モバイルでも同等の UX を実現する必要があった。

改修前
BarTouchData(enabled: false) — タッチ無効
棒グラフをタップしても何も起きない
内訳確認には凡例の色から目視で読み取る必要があった
→
改修後
BarTouchData(enabled: true) — タッチ有効化
棒グラフをタップするとツールチップが表示される
メンバー名 + 件数が一覧で表示される（0件はスキップ）
棒グラフ（タップ後のイメージ）
タップした棒（選択状態）

AC3
件数が0のメンバーはツールチップに表示されない

buildTooltipLines() 内で count > 0 の場合のみ行を追加する。count == 0 のメンバーはループをスキップし、ツールチップには含まれない。

AC4
未割当タスクがある場合、未割当の件数も表示される

day.countsByAssignee[null]（未割当の件数）が 1 件以上の場合、メンバー行の後ろに「未割り当て: N」を追加する。null キーが未割当を表すのは既存の DailyOverview モデルの設計に準拠している。

実装メモ（仕様外の判断・変更・妥協点）

ツールチップ生成ロジックを overview_tooltip_builder.dart に切り出し

BarTouchTooltipData の getTooltipItem コールバック内にロジックを埋め込むとウィジェットテストが困難なため、buildTooltipLines() 関数として独立ファイルに切り出した。l10n に依存しない（文字列を引数で受け取る）純粋関数のため、ウィジェットなしで単体テストを書ける。

_OverviewChart に members プロパティを追加

ツールチップ生成時にメンバー名と nickname 参照が必要なため、従来の memberColorMap に加えて members: List<HouseholdMember> プロパティを追加した。
