## #57: アナウンスマスタメンテ画面の追加（管理画面配下）

**GitHub Issue**: ryokkon624/hw-hub-manage#57
**ブランチ**: feature/57-admin-announcements（新規）
**コミット参照**: (ryokkon624/hw-hub-manage#57)

### ユーザーストーリー

**As a** システム管理者
**I want to** アナウンスを管理画面から登録・編集・削除したい
**So that** 開発者の手を借りずにアナウンスの内容や表示期間を管理できる

### Acceptance Criteria

- [x] AC1: 管理画面配下（`/admin/announcements`）にアナウンス一覧画面が追加される
- [x] AC2: 一覧にはタイトル（表示言語に応じた値）・重要度・対象スコープ・有効期間・ステータス（有効中 / 期限切れ）が表示される
- [x] AC3: 新規登録フォームから、タイトル（ja/en/es）・本文（ja/en/es）・重要度（m_code 0028）・対象スコープ（m_code 0027）・有効開始日時・有効終了日時を入力して登録できる
- [x] AC4: 既存アナウンスを編集・削除できる
- [x] AC5: バックエンドに管理者向け CRUDエンドポイント（`POST /api/admin/announcements`・`PUT /api/admin/announcements/{id}`・`DELETE /api/admin/announcements/{id}`）が実装される

### 備考

- 優先順位の根拠：アナウンスバナー機能Issueの後続実装
- 依存関係：アナウンスバナー機能Issue（前提）
- 対象リポジトリ: hw-hub-frontend / hw-hub-backend（両方）

---

## リスク

- フロントエンド（Vue 3）とバックエンド（Spring Boot）の両方に実装が必要で、開発範囲が広い
- バックエンドの管理者向けCRUDエンドポイントは管理者権限チェック（認可）が必要
- m_code 0028（重要度）・m_code 0027（対象スコープ）のマスタデータ参照方式を既存実装（`/api/codes`）に合わせる必要がある
- フロントエンドの管理画面配下のルーティング・レイアウト構成を既存 `/admin` 配下の実装パターンに揃える必要がある
