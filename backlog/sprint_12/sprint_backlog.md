## Issue #34: [SP版]買い物リスト画面の3フレームをタブ構成に変更したい

**GitHub Issue**: ryokkon624/hw-hub-manage#34
**Story Points**: 5
**ブランチ**: feature/34-shopping-list-tab-layout
**対象リポジトリ**: hw-hub-frontend

### ユーザーストーリー

As a スマートフォンでHwHubを使うユーザー
I want to 買い物リスト画面をタブで切り替えられるようにしたい
So that 小さい画面でもステータス別の情報を整理して確認できる

### Acceptance Criteria

- [x] AC1: SP版の買い物リスト画面に「未購入」「かご」「購入済み」の3タブが表示される
- [x] AC2: 「未購入」タブには未購入アイテム一覧と購入場所フィルタが表示される
- [x] AC3: 「かご」タブにはかごアイテム一覧と購入済みにするボタンが表示される
- [x] AC4: 「購入済み」タブには直近7日間の購入済みアイテムが日付グループで表示される
- [x] AC5: 「未購入」「かご」タブのタブヘッダーに件数バッジが表示される
- [x] AC6: デフォルト表示は「未購入」タブ
- [x] AC7: PC版の表示は変更しない

### 注意事項

- SP版のみの変更。PC版レイアウトに影響しないこと
- 既存の3フレーム構成（未購入・かご・購入済み）の機能はそのままタブに移植する

## Issue #33: ユーザアイコンが取得できない場合にbroken imageではなく、イニシャルアイコンを表示したい

**GitHub Issue**: ryokkon624/hw-hub-manage#33
**Story Points**: 2
**ブランチ**: fix/33-user-icon-fallback
**対象リポジトリ**: hw-hub-frontend

### ユーザーストーリー

As a HwHubを使うユーザー
I want to プロフィール画像が取得できない場合でもイニシャルアイコンが表示されるようにしたい
So that broken imageが表示されて画面が崩れる状態を解消できる

### Acceptance Criteria

- [x] AC1: imgタグのエラーイベント（画像取得失敗）が発生した場合、プロフィール画像未登録ユーザーと同じイニシャルアイコンを表示する
- [x] AC2: イニシャルアイコンの外観（背景色・文字色・サイズ）は画像未登録ユーザーの表示と統一されている
- [x] AC3: アイコンが使われているすべての箇所で画像取得失敗時にイニシャルアイコンを表示する。ただしアカウント設定画面のプロフィール画像は例外で、画像取得失敗時も画像未登録時と同じ表示（イニシャルアイコンではない）とする

### 注意事項

- アカウント設定画面のプロフィール画像はAC3の例外。壊れた画像表示ではなく「画像未登録時と同じ表示」を維持する
- 画像未登録ユーザーとの表示統一が重要。既存のイニシャルアイコン実装を再利用すること
