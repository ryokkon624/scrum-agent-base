Sprint 60 Review

#173 — Notifier Provider の定義が各 Notifier ファイルに分散している

Sprint 60 #173 refactor mobile

なぜこの改修が必要だったか

問題

Riverpod の Provider 定義は {feature}_providers.dart に集約する規約だが、大半の feature で各 *_notifier.dart ファイル末尾に直接書かれており分散していた。Provider を探すために各 Notifier ファイルを個別に開く必要があり、追加・変更・参照のコストが高かった。唯一 app_info feature だけが規約通りのパターンで実装されていた。

改修前（Provider が Notifier ファイルに混在）
my_tasks_notifier.dart 末尾に myTasksNotifierProvider
shopping_list_notifier.dart 末尾に shoppingListNotifierProvider
login_notifier.dart 末尾に loginNotifierProvider
Provider を調べるにはどのファイルに書かれているか知っている必要がある
→
改修後（{feature}_providers.dart に集約）
my_tasks_providers.dart に myTasksNotifierProvider を移動
shopping_providers.dart に3つの Notifier Provider を集約
auth_providers.dart に7つの Notifier Provider を集約
feature の Provider は providers.dart の1ファイルを見るだけで把握できる
