Sprint 60 Review

#174 — 単一ページの feature で presentation 配下に不要なサブフォルダがある

Sprint 60 #174 refactor mobile

なぜこの改修が必要だったか

問題

画面が1つしかない feature（account_settings / household_settings / settings）で、presentation/{feature_name}/ という不要なサブフォルダが1層余分に存在していた。複数画面 feature の命名規則（presentation/{screen_name}/）が、単一画面 feature にも誤って適用されたことが原因。ファイルがどの階層にあるか直感的にわからず、毎回余分なフォルダを開く手間があった。

改修前（不要な中間フォルダあり）
account_settings/presentation/account_settings/account_settings_page.dart
household_settings/presentation/household_settings/household_settings_page.dart
settings/presentation/settings_top/settings_top_page.dart
単一画面なのに3階層（feature/presentation/subfolder/file）
→
改修後（2階層に統一）
account_settings/presentation/account_settings_page.dart
household_settings/presentation/household_settings_page.dart
settings/presentation/settings_top_page.dart
単一画面 feature は2階層（feature/presentation/file）に統一
