## Issue #121: [mobile] アカウント設定画面を実装する（画面#18）

### ユーザーストーリー

**As a** ユーザー
**I want to** プロフィール・パスワード・アイコン・通知設定を自分で管理したい
**So that** アプリの利用環境を自分の好みに合わせられる

### Acceptance Criteria

- [x] AC1: メール・表示名が読み取り専用で表示される
- [x] AC2: パスワード変更（現在/新規/確認）が可能。Googleのみアカウントの場合はセクション非表示
- [x] AC3: 表示名（必須・最大50文字）と言語（ja/en/es）を変更でき、変更があるときのみ「保存」ボタンが活性になる
- [x] AC4: image_picker でカメラ/フォトライブラリからアイコン画像を変更できる（5MB以内・JPEG/PNG/GIF、S3 Presigned URL方式）
- [x] AC5: 通知設定（グローバルON/OFF・世帯関連・タスク割り当て）をトグルで即時反映できる
- [x] AC6: メールが @gmail.com のユーザーにのみ Google 連携セクションが表示され、未連携/連携済みで表示が切り替わる
- [x] AC7: アカウント削除は確認ダイアログ後に実行。他の有効メンバーが存在する世帯の OWNER は削除不可でエラー表示

## リスク・チャレンジ

### リスク

- AC4: image_picker + S3 Presigned URL方式はモバイル実装として複雑度が高い。カメラ/ライブラリ両対応、5MB制限のバリデーション、JPEG/PNG/GIF判定を漏れなく実装する必要がある
- AC2: Googleのみアカウントの判定ロジック（認証プロバイダー種別の取得）をバックエンドと連携して正確に実装する必要がある
- AC7: OWNER削除不可の判定はサーバーサイドと連携。エラーハンドリングのUIを適切に実装する

### チャレンジ

- AC3〜AC5は変更検知ロジック（dirty state管理）が複数セクションに渡るため、Riverpodの状態管理設計を慎重に行う
- AC6のGoogle連携セクションは未連携/連携済みで表示が切り替わる動的UIを実装する
