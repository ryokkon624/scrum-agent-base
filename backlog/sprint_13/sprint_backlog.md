## Issue #42: 買い物リスト画面にスワイプジェスチャーを導入したい

**GitHub Issue**: ryokkon624/hw-hub-manage#42
**Story Points**: 5
**ブランチ**: feature/42-swipe-gesture
**対象リポジトリ**: hw-hub-frontend

### ユーザーストーリー

As a スマートフォンで買い物リストを使うユーザー
I want to アイテムをスワイプして操作したい
So that 片手でも直感的にアイテムの状態を変更でき、スムーズに買い物ができる

### スワイプ操作仕様

| 状態 | 左スワイプ（←） | 右スワイプ（→） |
|------|---------------|---------------|
| 未購入 | 削除（背景：赤 / trash-2アイコン / 確認ダイアログ表示） | かごへ移動（背景：緑 / shopping-cartアイコン / 即時移動） |
| かご | 未購入へ戻す（背景：グレー / list-restartアイコン / 即時戻る） | 購入済みにする（背景：緑 / walletアイコン / 即時移動） |
| 購入済み | メッセージ表示（「編集画面で更新してください」） | なし |

### Acceptance Criteria

- [x] AC1: 未購入アイテムを右スワイプするとかごへ即時移動する（背景：緑 / shopping-cartアイコン）
- [x] AC2: 未購入アイテムを左スワイプすると削除確認ダイアログが表示され、確認後に削除される（背景：赤 / trash-2アイコン）
- [x] AC3: かごのアイテムを左スワイプすると未購入へ即時戻る（背景：グレー / list-restartアイコン）
- [x] AC4: かごのアイテムを右スワイプすると購入済みへ即時移動する（背景：緑 / walletアイコン）
- [x] AC5: 購入済みアイテムを左スワイプすると「編集画面で更新してください」メッセージが表示される
- [x] AC6: 指のドラッグに合わせてアイテムが左右にスライドし、画面幅の30%以上スワイプして指を離すとアクションが確定する
- [x] AC7: スワイプ操作はSP版（モバイル幅）でのみ有効で、PC版はボタン操作のまま

### 注意事項

- スワイプはSP版（モバイル幅）のみ有効。PC版の既存ボタン操作に影響しないこと
- 指のドラッグに合わせてアイテムがスライドするアニメーションが必要
- アクション確定閾値は画面幅の30%以上
