原因分析

🔍 表示用URLへのLocalhostアドレス変換が適用されていなかった

開発環境ではLocalStackを使用しており、バックエンドがS3のURLとして http://localhost:4566/... を返す。 Androidエミュレーターから localhost は到達不能（= ホストPCではなくエミュレーター自身を指す）のため、 画像ロードが失敗してイニシャル表示にフォールバックしていた。

アップロード用のPresigned URLには変換が存在していたが（AccountSettingsRepositoryImpl._resolveS3Url()）、 プロフィール取得時（fetchProfile / getMyProfile）の戻り値には変換が適用されていなかったのが根本原因。

実装メモ（仕様外の判断・変更・妥協点）

判断1

S3UrlResolver をシングルトン const クラスで実装
isDebug を外から注入する設計にすることで、テストで const S3UrlResolver(isDebug: true) として使いやすくなった。Providerからは S3UrlResolver(isDebug: kDebugMode) として毎回生成する。

判断2

AccountSettingsRepositoryImpl は positional パラメータ（既存パターン踏襲）
AuthRepositoryImpl は named パラメータだが、AccountSettingsRepositoryImpl は既存コードが (Dio, Dio) の positional 形式だったため、3番目に S3UrlResolver を positional で追加して既存パターンに合わせた。

判断3

UserProfileDto に copyWith がないため全フィールドを明示して新インスタンスを生成
UserProfileDto は const コンストラクタを持つ不変 DTO で copyWith が未定義。変換後の iconUrl だけ差し替えるために全フィールドを明示して新しいインスタンスを生成した。
