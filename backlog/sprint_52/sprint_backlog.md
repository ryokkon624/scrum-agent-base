## Issue #126 — [mobile] プロフィール画像を登録済みのユーザーのuser_iconに画像が表示されない

### 発生事象

プロフィール画像を登録しているユーザーでも、ヘッダーのユーザーアイコン（user_icon）に画像が表示されず、イニシャル文字が表示される。

### ユーザーストーリー

**As a** HwHubモバイルアプリのユーザー
**I want to** 登録したプロフィール画像がヘッダーのユーザーアイコンに表示されるようにしたい
**So that** 自分のアカウントを視覚的に識別できる

### Acceptance Criteria

- [x] AC1: プロフィール画像を登録済みのユーザーのヘッダーアイコンに、登録した画像が円形で表示される
- [x] AC2: プロフィール画像が未登録のユーザーは従来通りdisplay_nameの先頭2文字が表示される

### 備考

- 対象: ヘッダーのユーザーアイコン（HeaderUserIcon ウィジェット）
- Sprint 44 Sprint Reviewで発覚
- 同症状の修正は過去にも実施済み（2回目）。
- バックエンドが返すS3 URLをモバイル側で変換する処理が存在するため、その箇所を重点的に調査すること。
- 開発環境ではLocalStackを使用しているためバックエンドが返すURLが `localhost` になっており、それをPC（ホスト）のアドレスに変換する処理がどこかにあるはず。該当箇所に問題が混入している可能性がある。
- **bugラベル → 計画フェーズで根本原因調査・改修方針を整理し、IssueのBodyを更新すること**
- ブランチ: `fix/126-mobile-user-icon-image-not-displayed`
- コミット参照: `(ryokkon624/hw-hub-manage#126)`

## Issue #127 — [mobile] アカウント設定画面でプロフィール画像登録後に画面の画像が更新されない

### 発生事象

アカウント設定画面でプロフィール画像を登録すると、DBへの登録自体は正常に完了しているが、画面上のプロフィール画像が更新されない。画像の取得（再フェッチ）ができていないと考えられる。

### ユーザーストーリー

**As a** HwHubモバイルアプリのユーザー
**I want to** アカウント設定画面でプロフィール画像を変更した後、変更後の画像が即座に表示されてほしい
**So that** 設定変更の結果を画面上で確認できる

### Acceptance Criteria

- [x] AC1: アカウント設定画面でプロフィール画像を変更した後、画面上のプロフィール画像が変更後の画像に更新される
- [x] AC2: ヘッダーのユーザーアイコンにも変更後の画像が反映される

### 備考

- 対象: アカウント設定画面 icon_section / AccountSettingsNotifier
- Sprint 45 Sprint Reviewで発覚
- 依存関係: #126（プロフィール画像を登録済みのuser_iconに画像が表示されない）。#126の修正で解決する可能性があるが、アカウント設定画面での再フェッチ処理は独立した問題である可能性もあるため、別Issueで管理する。
- **bugラベル → 計画フェーズで根本原因調査・改修方針を整理し、IssueのBodyを更新すること**
- **りょこさんコメント**: #126が解消すれば#127はおのずと解消される可能性もある
- ブランチ: `fix/127-mobile-account-settings-icon-not-refreshed`
- コミット参照: `(ryokkon624/hw-hub-manage#127)`

## Issue #130 — [mobile] アカウント設定画面に外観設定セクション（システム連動/ライト/ダーク）を追加する

### ユーザーストーリー

**As a** HwHubモバイルアプリのユーザー
**I want to** アカウント設定画面でアプリの外観（ライト/ダーク/システム連動）を選択したい
**So that** 自分の好みや環境に合わせてアプリの見た目を変えられる

### Acceptance Criteria

- [x] AC1: プロフィール設定セクションとプロフィール画像セクションの間に外観セクションを配置する（WEB版と同様の配置）

- [x] AC4: アプリを再起動しても選択した外観設定が保持される（デバイスローカルに保存）
- [x] AC5（Nice to Have）: m_userの外観設定カラムと連動し、WEB版の設定と同期される

### 備考

- Sprint 45 Sprint Reviewでりょこさんより要望
- 依存関係：なし
- ブランチ: `feature/130-mobile-appearance-settings`
