実装メモ（仕様外の判断・変更・妥協点）

判断1（名前衝突の解決）

Flutter material の ThemeMode と HwHub 生成済み enum ThemeMode の名前衝突
lib/core/models/theme_mode.dart に HwHub 独自の ThemeMode enum が存在し、Flutter material の ThemeMode と名前が衝突する。 import 'package:flutter/material.dart' as material; と import '../models/theme_mode.dart' as hw; のエイリアスを使って解決した。

判断2（ThemeModeNotifier を AsyncNotifier で実装）

SharedPreferences の読み取りは非同期のため AsyncNotifier<material.ThemeMode> を採用
main.dart では AsyncData の valueOrNull ?? ThemeMode.system でフォールバックを設定している（初期化中も ThemeMode.system が適用される）。

判断3（AC5 の例外処理）

バックエンド同期失敗を UI に表示しないために例外的に catch (_) {} を採用
通常 catch (_) {}（握りつぶし）は禁止だが、AC5 は明示的に "Nice to Have" であり、失敗通知がむしろユーザー体験を悪化させる。デバイスローカルの設定（ThemeModeNotifier）は常に成功するため、実害はない。

判断4（AppearanceSection のウィジェットテスト追加）

convention-reviewerの指摘を受けて AppearanceSection のウィジェットテストを追加した
SegmentedButton の初期選択確認・onSelectionChanged からの setThemeMode() 呼び出し確認・選択後の state 更新を検証するテストを追加した。_FakeThemeModeNotifier を使って capturedMode で呼び出し確認を行う設計。
