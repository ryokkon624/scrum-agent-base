原因分析

🔍 2つのバグが同居（go_router の context ミスマッチ + isNotPurchased getter の誤条件）

バグ1: go_router 環境での Navigator.pop context ミスマッチ

_confirmDeleteItem 内の AlertDialog で builder: (_) => とダイアログの BuildContext を捨て、actions の Navigator.pop(context, ...) を外側 (_DetailBodyState) の context で呼んでいた。go_router 環境では外側 context の Navigator.pop を呼ぶと、ダイアログではなく詳細画面ルート自体が pop されてしまう（「メイン画面が買い物リストに遷移」）。ダイアログは閉じないため showDialog の await が完了せず、deleteItem が呼ばれない。

バグ2: isNotPurchased getter にかご状態が含まれていた（AC3違反）

ステータス isNotPurchased（修正前） isNotPurchased（修正後） 削除ボタンの正しい状態
未購入（status='0'） true true 表示
かご（status='1'） true（バグ） false 非表示
購入済み（status='9'） false false 非表示
