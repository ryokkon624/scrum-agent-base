#96 — 買い物アイテム詳細画面で画像の削除をした際に
モーダルが消えず、画像も削除されない

Sprint 35 #96 bug mobile

発生事象

修正前（バグあり）
詳細画面 - 添付画像
IMG ×
この画像を削除しますか？
キャンセル この画像を削除
→「この画像を削除」をタップすると詳細画面が閉じる（go_router が詳細ルートをpop）。モーダルは消えず、画像も削除されない。
修正後（正常）
詳細画面 - 添付画像（削除後）
（画像なし）
「この画像を削除」タップ → モーダルが閉じる → deleteAttachment が呼ばれ画像が削除される → 詳細画面の画像一覧からサムネイルが消える
→ モーダルが正常に閉じ、画像も削除される。詳細画面はそのまま表示され続ける。

原因分析

🔍 #94 と同一根本原因：go_router 環境での AlertDialog context ミスマッチ

_confirmDeleteAttachment 内の showDialog の builder: (_) => でダイアログ専用の BuildContext を捨て、Navigator.pop(context, ...) に外側 (_DetailBodyState) の context を使っていた。

go_router 環境では showDialog が内部的に Navigator スタックの上に重ねるため、外側の context からは「ルートのポップ」として解釈される。これにより：

修正前のフロー（バグあり）
× 「この画像を削除」タップ → Navigator.pop(context, true)
→ 外側context = _DetailBodyState の context（詳細画面ルート）
→ go_router が詳細画面をpop（メイン画面の買い物リストへ遷移）
→ ダイアログは閉じず、showDialog の await が完了しない
→ deleteAttachment が呼ばれない（画像削除されない）
修正後のフロー（正常）
✓ 「この画像を削除」タップ → Navigator.pop(dialogContext, true)
→ dialogContext = ダイアログ専用の BuildContext
→ ダイアログだけが閉じる
→ showDialog の await が true を返す
→ deleteAttachment(attachmentId) が呼ばれ画像が削除される

動作確認ポイント（りょこさん確認用）

  • 1
    画像サムネイルの「×」ボタンをタップすると確認ダイアログが表示されること 詳細画面の添付画像セクションでサムネイルの×ボタンをタップして「この画像を削除しますか？」が表示されるか確認
  • 2
    「この画像を削除」をタップするとダイアログが閉じ、詳細画面のままで画像が消えること 削除後に詳細画面が閉じず（買い物リストへの意図しない遷移なし）、該当画像のサムネイルが消えているか確認
  • 3
    「キャンセル」をタップするとダイアログが閉じ、画像が残っていること キャンセル後に詳細画面に戻り、画像が削除されていないことを確認
  • 4
    複数の画像がある場合、1枚だけ削除して他の画像が残ること 複数画像のうち1枚のみ削除し、残りの画像が引き続き表示されるか確認
