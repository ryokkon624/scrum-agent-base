#93 — 買い物アイテム追加画面でアイテム追加後、
買い物リスト画面に表示されない

Sprint 35 #93 bug mobile

発生事象

修正前（バグあり）
買い物リスト（未購入タブ）
牛乳 お気に入り
シャンプー なし
→ 「卵」を追加して戻ってきても表示されない。詳細画面でお気に入りにしても、一覧に反映されない。アプリ再起動後に反映される。
修正後（正常）
買い物リスト（未購入タブ）
牛乳 お気に入り
シャンプー お気に入り
卵（新規追加） なし
→ アイテム追加後に即時リストに表示。詳細画面でお気に入りにすると即時反映される。

原因分析

🔍 shoppingListNotifierProvider の invalidate が呼ばれていなかった

アイテム追加（createItem 成功）後・お気に入り操作（toggleFavorite）後に、一覧側の shoppingListNotifierProvider を invalidate していなかった。

ShoppingListNotifier は AutoDispose だが、StatefulShellRoute.indexedStack 配下では買い物リスト画面のウィジェットが IndexedStack で保持されるため、AutoDispose による破棄が起きない。そのため追加・お気に入り操作後も古い state が残り、変更が反映されない。

修正前：invalidate なし → 古いstateが残る
ShoppingItemNewPage → createItem成功 → invalidate なし → 古いstateのまま
修正後：invalidate → 次に表示された際に再fetch
ShoppingItemNewPage → createItem成功 → ref.invalidate(shoppingListNotifierProvider) → リスト画面に戻った際に再fetch・即時反映

再発防止

AutoDispose Provider と IndexedStack の組み合わせで「古いstateが残る」パターンは発生しやすい。invalidate による強制再取得パターンをテストで担保した。
