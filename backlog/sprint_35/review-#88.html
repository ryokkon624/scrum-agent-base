原因分析

🔍 purchasedItems getter の条件と _copyItemWithStatus の更新内容のミスマッチ

shopping_list_state.dart の purchasedItems getter は status == '9' かつ purchasedAt != null の両方を要求している。一方、shopping_list_notifier.dart の markPurchased / bulkPurchase は _copyItemWithStatus でステータスを '9' に変えるが、purchasedAt は元の値（最初は null）をそのまま保持していた。

そのため購入済みタブの getter フィルタを通らず画面に表示されない。未購入↔かごの操作は purchasedAt を条件にしないため即時反映される。アプリ再起動後はサーバーから purchasedAt 付きで返ってくるため表示される。

操作 status purchasedAt（修正前） purchasedItems getter の判定
スワイプ購入済み '9' null のまま → 除外（nullはfalse）
アプリ再起動後 '9' サーバーから返る → 表示される
