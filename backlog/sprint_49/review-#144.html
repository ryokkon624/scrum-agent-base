Sprint 49 Review

#144 — プライバシーポリシー画面を実装する

Sprint 49 #144 feature mobile

なぜこの実装が必要だったか

背景

アプリ情報画面（#142）から「プライバシーポリシー」リンクをタップしても、遷移先が _P('プライバシーポリシー') というプレースホルダーのままだった。個人情報の取り扱いについてユーザーが確認できるよう、6セクション構成のプライバシーポリシーを静的コンテンツとして実装する必要があった。

実装前
アプリ情報 → プライバシーポリシーをタップ
「プライバシーポリシー」テキストのみ表示
個人情報の取り扱いを確認不可
→
実装後
アプリ情報 → プライバシーポリシーをタップ
6セクション + 注記をスクロール表示
API コール不要（静的コンテンツ）
実装後の画面イメージ
← プライバシーポリシー
Housework Hub（以下「本サービス」）では、利用者の個人情報を適切に取り扱うため...
1. 収集する情報
• メールアドレス、表示名などのアカウント情報
• 世帯情報、家事テンプレート、買い物リスト等の利用データ
• アクセス日時、IP アドレス、ブラウザ情報などのログ情報
2. 情報の利用目的
• 本サービスの提供・維持・改善のため
3. 第三者提供
法令に基づく場合を除き、利用者の同意なく個人情報を第三者に提供することはありません。
... スクロールで続きを表示 ...
※ 実際に公開する際は、法令やストアポリシーに合わせて内容を調整し、専門家による確認を行うことをおすすめします。

AC別 実装内容

✓
AC1
6セクション（収集する情報・利用目的・第三者提供・安全管理措置・開示訂正削除・お問い合わせ）が i18n キーを使って表示される

各セクションを privacy* 系 ARB キーで定義。PrivacyPolicyPage では _Section ウィジェットで各項目を表示する。箇条書きが必要なセクション1・2では _BulletText で各項目を列挙し、セクション3〜6はテキストのみで表示する。

セクション Key 形式
1. 収集する情報 privacySection1 箇条書き 3項目
2. 情報の利用目的 privacySection2 箇条書き 3項目
3. 第三者提供 privacySection3 テキスト
4. 情報の安全管理 privacySection4 テキスト
5. 情報の保存期間 privacySection5 テキスト
6. お問い合わせ privacySection6 テキスト

    6セクションのプライバシーポリシーコンテンツが表示されること 「1. 収集する情報」「2. 情報の利用目的」「3. 第三者提供」「4. 情報の安全管理」「5. 情報の保存期間」「6. お問い合わせ」の各タイトルが見えること

    ページ末尾まで下スクロールできること コンテンツが画面を超えてスクロール可能であり、「6. お問い合わせ」の後に注記（「※ 実際に公開する際は...」）が表示されること
  • 4
    セクション1（収集する情報）とセクション2（利用目的）で箇条書きが表示されること セクション1はアカウント情報・利用データ・ログ情報の3項目、セクション2はサービス提供・トラブルシューティング・分析の3項目が箇条書きで表示されること
