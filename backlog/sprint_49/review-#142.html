GET /actuator/info で API バージョンを取得・表示し、取得中は「取得中...」・失敗時は「不明」が表示される

AppInfoRepository（interface + impl）を新規作成。GET /actuator/info のレスポンス {"app":{"version":"..."}} から version 文字列を取得する。Notifier 側では isLoadingApi フラグを使って取得中/完了/失敗の3状態を管理し、ウィジェット側でそれぞれ異なる表示を行う。

18+ final response = await _dio.get<Map<String, dynamic>>('/actuator/info');

20+ if (data == null) return null;

22+ if (app == null || app is! Map) return null;

💡 isLoadingApi フラグで「取得中（true）」「取得完了（false + apiVersion非null）」「失敗（false + apiVersionがnull）」の3状態を区別している。ウィジェット側では Key を付与した3種のウィジェット（apiVersionLoading・apiVersionValue・apiVersionUnknown）を条件分岐で切り替える。

動作確認ポイント（りょこさん確認用）

  • 1
    設定 → アプリ情報をタップすると5セクションの画面が表示されること バージョン・利用規約リンク・データとプライバシー・開発者情報・OSSライセンスの順でカード表示される
  • 2
    バージョンセクションで「UI」にアプリのバージョン番号が表示されること pubspec.yaml の version に設定された値（例: 1.0.0）が表示される
  • 3
    画面表示直後は「API」欄に「取得中...」が表示され、その後バージョン番号に切り替わること バックエンドが起動していない場合は「取得失敗」と表示される
  • 4
    「利用規約」をタップすると利用規約画面に遷移すること 戻るボタンでアプリ情報画面に戻れること（context.push を使用）
  • 5
    「プライバシーポリシー」をタップするとプライバシーポリシー画面に遷移すること 戻るボタンでアプリ情報画面に戻れること
