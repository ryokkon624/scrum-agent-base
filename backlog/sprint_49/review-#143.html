Sprint 49 Review

#143 — 利用規約画面を実装する

Sprint 49 #143 feature mobile

なぜこの実装が必要だったか

背景

アプリ情報画面（#142）から「利用規約」リンクをタップしても、遷移先が _P('利用規約') というプレースホルダーのままだった。サービス利用に関するルールをユーザーに提示するため、第1〜6条の利用規約コンテンツを静的コンテンツとして実装する必要があった。

実装前
アプリ情報 → 利用規約をタップ
「利用規約」テキストのみのプレースホルダー
規約コンテンツ表示不可
→
実装後
アプリ情報 → 利用規約をタップ
第1〜6条 + 注記をスクロール表示
API コール不要（静的コンテンツ）
実装後の画面イメージ
← 利用規約
この規約は、Housework Hub（以下「本サービス」）の利用条件を定めるものです...
第1条（適用）
本規約は、本サービスの提供条件および利用に関する権利義務関係を定めるものです。
第2条（アカウント）
• 利用者は、正確な情報を登録のうえ本サービスのアカウントを作成します。
• アカウント情報の管理は利用者自身が責任を負うものとします。
• 第三者による不正利用等が判明した場合は、速やかに運営者へ連絡してください。
第3条（禁止事項）...
... スクロールで続きを表示 ...
※ 実際に公開する際は、必要に応じて専門家による確認を行ってください。

AC別 実装内容

✓
AC1
第1〜6条の利用規約コンテンツが i18n キーを使って表示される

各条文を terms* 系 ARB キーで定義。TermsPage では _ArticleSection ウィジェットで各条を表示する。箇条書きが必要な第2〜3条では _BulletText で各項目を列挙する。

AC4
API コールを行わない（静的コンテンツのみ）

TermsPage は StatelessWidget で実装。Riverpod の ConsumerWidget ではなく通常の StatelessWidget を採用することで、API コールを行わない静的コンテンツ専用画面であることを型で明示している。

  • 1
    アプリ情報 → 「利用規約」タップで利用規約画面に遷移すること AppBar に「利用規約」と表示され、戻るボタン（«）でアプリ情報画面に戻れること
  • 2
    第1条〜第6条の利用規約コンテンツが表示されること 「第1条（適用）」「第2条（アカウント）」「第3条（禁止事項）」「第4条（サービスの変更・停止）」「第5条（免責事項）」「第6条（規約の変更）」の各タイトルが見えること
  • 3
    ページ末尾まで下スクロールできること コンテンツが画面を超えてスクロール可能であり、第6条の後に注記（「※ 実際に公開する際は...」）が表示されること
  • 4
    第2条と第3条で箇条書き（• ）が表示されること 第2条はアカウント管理に関する3項目、第3条は禁止事項4項目が箇条書きで列挙されること
