AC5
返信テキストが空の場合は送信ボタンが無効になる

AC4 の実装内容に内包。ValueListenableBuilder で _replyController の値を監視し、value.text.trim().isNotEmpty のときのみ onPressed に関数を設定（空のとき null → ボタン非活性）。

AC6
AI_ANSWERED 時に「解決した」「スタッフへ引き継ぐ」ボタンが表示される

canEscalate = status == InquiryStatus.aiAnswered で「スタッフへ引き継ぐ」ボタンの表示を制御。返信エリア内に条件分岐で表示し、タップすると _confirmEscalate() の確認ダイアログを経由して API を呼ぶ。「解決した」ボタンは CLOSED 以外で常に表示（AC7・AC8 と合わせて制御）。

AC8
各アクションボタンは確認ダイアログを経由して API を呼び出す

「解決した」「スタッフへ引き継ぐ」の両ボタンで showDialog を使った確認ダイアログを表示。キャンセルを選ぶとダイアログが閉じるだけ（API 呼び出しなし）、確認を選ぶと対応する API を呼ぶ。ダイアログの BuildContext は builder: (dialogContext) で明示的に受け取り、Navigator.pop(dialogContext) を使う（外側の context を使うと go_router でページが閉じるバグを防止）。

動作確認ポイント（りょこさん確認用）

  • 1
    問い合わせのメッセージスレッドがチャット形式で表示されること ユーザーのメッセージは右寄せ（緑バブル）、AI/スタッフのメッセージは左寄せ（白バブル）で表示されること
  • 2
    ステータスに応じた説明文がヘッダーに表示されること 例: AI_ANSWERED 状態で「AIが回答しました。解決した場合はクローズ...」のような説明文が表示されること
  • 3
    AI_ANSWERED 状態で「スタッフへ引き継ぐ」ボタンが表示されること 返信フォームの上に「スタッフへ引き継ぐ」ボタンが表示され、タップすると確認ダイアログが表示されること
  • 4
    STAFF_ANSWERED 状態で「解決した」ボタンのみが表示されること 「スタッフへ引き継ぐ」ボタンは非表示、「解決した」ボタンのみ表示されること
  • 5
    CLOSED 状態で返信フォームが非表示になること クローズ済みの問い合わせでは返信フォーム・「解決した」・「スタッフへ引き継ぐ」が一切表示されないこと
  • 6
    返信テキストが空のとき送信ボタンがグレー（無効）になること テキストを入力すると送信ボタンが有効になり、削除すると再び無効になること
  • 7
    「解決した」「スタッフへ引き継ぐ」ボタンで確認ダイアログが表示されること ダイアログでキャンセルを選ぶと操作がキャンセルされ、確認を選ぶと API が呼ばれスナックバーが表示されること
