AC別 実装内容

✓
AC1
カテゴリ・件名・お問い合わせ内容の必須バリデーションが動作する

InquiryCreateNotifier.submit() が「送信」ボタン押下時に呼ばれ、バリデーションエラーがあれば errorMessage を i18n キーで state にセット。Page 側で対応するフィールド下に ARB のエラーメッセージを表示する。

チェック内容 エラーキー（ARB） 表示場所
カテゴリ未選択 inquiryCreateErrorCategoryRequired カテゴリドロップダウン下
件名が空 inquiryCreateErrorTitleRequired 件名フィールド下
件名が200文字超（AC2） inquiryCreateErrorTitleTooLong 件名フィールド下
内容が空 inquiryCreateErrorBodyRequired 内容フィールド下

✓
AC2
件名が200文字を超えるとバリデーションエラーが表示される

state.title.length > 200 の判定でエラーキー inquiryCreateErrorTitleTooLong を state にセット。Page 側で件名フィールド下に ARB の日本語エラーメッセージが表示される。

💡 AC1（必須チェック）と AC2（文字数チェック）を別々の errorMessage キーで管理。件名フィールド下では if / else if で両方のエラーを表示できるようにしている（両方同時には出ない設計）。
✓
AC3
POST /api/inquiries で送信され、成功後スナックバーが表示される

バリデーション通過後に InquiryRepositoryImpl.createInquiry() が POST /api/inquiries を呼び出す。成功すると API が返す {"inquiryId": N} から ID を取得して state にセット。Page 側の ref.listen が createdInquiryId の変化を検知し、スナックバーを表示する。

AC4
送信成功後、作成した問い合わせの詳細画面へ遷移する

AC3 の ref.listen で createdInquiryId の変化を検知し、context.push(AppRoutes.settingsInquiryDetail(id)) を呼んで詳細画面へ push する。スタックに一覧→新規作成→詳細が積まれる状態になる。

動作確認ポイント（りょこさん確認用）

  • 1
    「送信」ボタンを押すとバリデーションエラーが表示されること（未入力） カテゴリ未選択・件名未入力・内容未入力の状態で送信ボタンを押すと、エラーになったフィールドの下に赤いエラーメッセージが表示されること
  • 2
    件名が201文字以上のときに文字数エラーが表示されること 件名を200文字を超えて入力して送信ボタンを押すと「件名は200文字以内で入力してください」のようなエラーが表示されること
  • 3
    正常送信後にスナックバーが表示されて詳細画面へ遷移すること カテゴリ・件名・内容を正しく入力して送信すると「送信しました」のスナックバーが表示され、作成した問い合わせの詳細画面に自動的に遷移すること
  • 4
    「キャンセル」ボタンで問い合わせ一覧に戻ること キャンセルボタンをタップすると問い合わせ一覧画面に戻ること（詳細画面へは遷移しないこと）
  • 5
    送信中は送信ボタンが非活性（スピナー表示）になること 送信ボタンを押した後、APIレスポンスが返るまでの間はボタンがグレーになりスピナーが表示されること
