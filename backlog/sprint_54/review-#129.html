原因分析

🔍 GoogleSignIn の serverClientId 未設定 + エラー握りつぶし

google_link_section.dart の _handleLink で GoogleSignIn を初期化する際、serverClientId（バックエンドのGoogle Web Client ID）が指定されていなかった。google_sign_in v6 系では serverClientId を渡さないと、取得される idToken の aud がモバイルアプリのクライアントIDになり、バックエンドの tokeninfo 検証で期待する sub と一致しないため Google 連携 API が失敗する。

さらに on Exception catch (_) でエラーを握りつぶしていたため、ユーザーには成功したように見えてしまい、結果として m_user.auth_provider が更新されないまま画面のセクション表示も切り替わらなかった。

実装メモ（仕様外の判断・変更・妥協点）

AppConfig.googleServerClientId のデフォルト値を空文字列に設定: 開発環境では flutter run --dart-define=GOOGLE_SERVER_CLIENT_ID=xxx で設定する想定。空の場合は null を渡して従来通りの動作にフォールバックする設計にした。
GoogleLinkSection を StatefulWidget に変換: エラーハンドリングで mounted チェックと context 参照が必要なため StatefulWidget が必要。_errorMessage などの State 管理は不要で、AppSnackBar でエラー表示するためシンプルなまま。
GoogleSignIn の _handleLink のウィジェットテスト: Google sign-in 自体はプラットフォームプラグインのためウィジェットテストでは直接検証できない。既存テスト（表示確認）は引き続き通ることを確認した。

AC確認

✓
AC1
Google連携後にm_user.auth_provider / auth_provider_id が正しく更新される
serverClientId を設定したことで、バックエンドの tokeninfo 検証が成功し、POST /api/users/me/google/link/mobile が正常に処理されて DB が更新されるようになった。
✓
AC2
Google連携完了時に即座にパスワード変更セクションが非表示になる
API が成功すると Notifier が state を更新し（isLinked: true）、パスワード変更セクションの表示条件（!state.isGoogleLinked）を満たさなくなるため非表示に切り替わる。
✓
AC3
Google連携完了時に即座にGoogle連携セクションが「連携済み」表示に切り替わる
GoogleLinkSection の widget.isLinked が true になると _buildLinked() が呼ばれ、「連携済み」バッジが表示される。Notifier の state 更新と連動して即座に反映される。
