## Issue #129 — アカウント設定画面でGoogle連携後にm_userのauth_providerが更新されない

**ラベル**: bug  
**ブランチ**: `fix/129-mobile-account-settings-google-link-not-updating-user`  
**リポジトリ**: hw-hub-mobile

### 発生事象

アカウント設定画面でGoogle連携を実行すると連携自体はできたように見えるが、以下の問題が発生している。
- アカウント設定画面のパスワード変更セクションが引き続き表示される（Googleのみアカウントは非表示のはず）
- Google連携セクションが引き続き表示される（連携済みなら切り替わるはず）
- m_user.auth_provider および auth_provider_id が更新されていない

### ユーザーストーリー

**As a** HwHubモバイルアプリのユーザー  
**I want to** Google連携後にアカウント設定画面の表示が連携済み状態に切り替わってほしい  
**So that** 連携が正常に完了したことを確認でき、以後パスワード変更セクションが非表示になる

### Acceptance Criteria

- [x] AC1: Google連携後にm_user.auth_provider / auth_provider_id が正しく更新される
- [x] AC2: Google連携完了時に即座にパスワード変更セクションが非表示になる
- [x] AC3: Google連携完了時に即座にGoogle連携セクションが「連携済み」表示に切り替わる

### 原因

（計画フェーズで調査・更新）

### 改修方針

（計画フェーズで調査・更新）

### 備考

- 対象: アカウント設定画面 google_link_section / POST /api/users/me/google/link/mobile
- Sprint 45 Sprint Reviewで発覚

## Issue #146 — 家事登録画面でテンプレート選択時に家事名が画面に反映されない

**ラベル**: bug  
**ブランチ**: `fix/146-mobile-chore-register-template-name-not-reflected`（計画フェーズで確定）  
**リポジトリ**: hw-hub-mobile

### 発生事象

家事登録画面でテンプレートから選択した際に、家事名がフォームに反映されない。画面上は空欄のままだが、登録ボタンを押下するとDBには家事名が登録されている。

### 再現手順

1. 家事登録画面を開く
2. 「テンプレートから選択」をタップしてモーダルを開く
3. テンプレートを選択する
4. フォームの家事名欄が空欄のままであることを確認する
5. 登録ボタンを押下する → DBには家事名が登録される

### ユーザーストーリー

**As a** HwHubモバイルアプリのユーザー  
**I want to** テンプレートを選択した際に家事名・説明欄が画面に反映されてほしい  
**So that** 入力内容と登録内容が一致していることを確認してから登録できる

### Acceptance Criteria

- [x] AC1: テンプレートを選択した後、家事名フィールドに選択したテンプレートの家事名が即座に反映される
- [x] AC2: テンプレートを選択した後、説明欄（description）にも選択したテンプレートの説明が即座に反映される

### 原因

（計画フェーズで調査・更新）

### 改修方針

（計画フェーズで調査・更新）

### 備考

- Sprint 47 Sprint Reviewで発覚（ユーザーの実機確認）

## Issue #147 — 家事編集画面で存在しない日付を入力しても保存ボタンが押下できる

**ラベル**: bug  
**ブランチ**: `fix/147-mobile-chore-edit-invalid-date-validation`（計画フェーズで確定）  
**リポジトリ**: hw-hub-mobile

### 発生事象

家事編集画面で存在しない日付（例: 2月30日など）を入力した場合でも、バリデーションが行われず保存ボタンが押下できる状態になっている。

### 再現手順

1. 家事編集画面を開く
2. 月次（MONTHLY）周期を選択し、日付欄に存在しない日付（例: 2月30日）を入力する
3. 保存ボタンが押下可能なままになっている

### ユーザーストーリー

**As a** HwHubモバイルアプリのユーザー  
**I want to** 存在しない日付を入力した際にエラーが表示されてほしい  
**So that** 誤った日付で家事が登録・更新されることを防げる

### Acceptance Criteria

- [x] AC1: 家事編集画面で存在しない日付（例: 2月30日）を入力した場合、バリデーションエラーが表示されて保存ボタンが非活性になる
- [x] AC2: 家事新規作成画面（#137）でも同様に存在しない日付入力時にバリデーションエラーが表示されて保存ボタンが非活性になる

### 原因

（計画フェーズで調査・更新）

### 改修方針

（計画フェーズで調査・更新）

### 備考

- Sprint 47 Sprint Reviewで発覚（ユーザーの実機確認）
