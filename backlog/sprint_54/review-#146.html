原因分析

🔍 TextFormField の initialValue はウィジェット初期化時にしか反映されない

HouseworkForm（旧 StatelessWidget）の TextFormField に initialValue: form.name を指定していた。initialValue は Flutter の内部 TextEditingController を最初の build 時にのみ初期化するため、テンプレート選択後に Notifier が form.name を更新しても TextFormField の表示は変わらない。

1
ユーザーがテンプレートを選択
2
Notifier の state.form.name が更新される → HouseworkForm が rebuild
3
initialValue: form.name は初回 build のみ有効 → 内部 Controller は初期値のまま（空文字）
4
画面は空欄のまま。しかし保存時は State の値を使うので DB には正しく登録される

修正方針: HouseworkForm を StatefulWidget に変換し、独自の TextEditingController を管理する。didUpdateWidget で外部からの form.name 変更を検知して _nameController.text に反映する。

この修正は家事編集画面（#147/#138 で実装済みの HouseworkEditPage）でも同じ HouseworkForm を使っているため、編集画面にも自動的に適用される。

再発防止

今回の根本原因は「TextFormField の initialValue が初回 build 時にしか反映されない Flutter の制約を知らずに実装した」こと。TextEditingController への反映漏れ防止ルールが mobile-conventions および long_term.md に記録済み（Sprint 47 追記）。
