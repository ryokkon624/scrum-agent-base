#145 — 家事設定画面で家事が0件の場合に
「読み込み中」と表示され続ける

Sprint 54 #145 bug mobile

発生事象

修正前（バグあり）
読み込み中
※ API は完了しており実際には0件
修正後（正常）
📄
家事がまだ登録されていません

カテゴリフィルタ選択中に0件になった場合は「フィルタに該当する家事がありません」と表示される。

原因分析

🔍 data（0件）を loading と同一の表示で扱っていた

housework_list_page.dart の _HouseworkListBody で state.filteredHouseworks.isEmpty の場合に l10n.commonLoading（「読み込み中」）の Text を表示していた。AsyncValue.data のコールバック内に入った時点でローディングは完了しているにもかかわらず、data（0件） と loading を区別せずに同じ表示をしていた。

状態 修正前の表示（問題） 修正後の表示（正しい）
loading CircularProgressIndicator ✓ CircularProgressIndicator（変更なし）
data（0件・フィルタなし） 「読み込み中」Text（誤り） → 「家事がまだ登録されていません」EmptyState
data（0件・フィルタあり） 「読み込み中」Text（誤り） → 「フィルタに該当する家事がありません」EmptyState
data（1件以上） ListView（正常） ✓ ListView（変更なし）

AC確認

✓
AC1
家事データが0件の場合、家事設定一覧画面にEmptyState（空状態）が表示される
filteredHouseworks.isEmpty かつ selectedCategory == null のとき、「家事がまだ登録されていません」の EmptyState テキスト（Key: houseworkSettingsEmptyAll）を表示する実装に修正した。カテゴリフィルタ選択中は「フィルタに該当する家事がありません」（Key: houseworkSettingsEmptyFiltered）を表示。
✓
AC2
「読み込み中」インジケーターが表示され続けない
AsyncValue.data コールバック内では commonLoading を表示しなくなった。ローディング中は AsyncValue.loading コールバックの CircularProgressIndicator が担当し、データ取得完了後は 0件/1件以上 を区別して正しい UI を表示する。

動作確認ポイント（ユーザー確認用）

  • 1
    家事が0件の世帯で家事設定画面を開く 「読み込み中」ではなく「家事がまだ登録されていません」が表示されることを確認
  • 2
    カテゴリフィルタを選択して家事が0件になった場合 「フィルタに該当する家事がありません」が表示されることを確認（例: CLEANING のみ家事があり COOKING フィルタを選択）
  • 3
    家事が1件以上ある世帯では従来通り一覧が表示されること EmptyState への変更で既存の一覧表示がデグレしていないことを確認

再発防止

今回の根本原因は AsyncValue.data が 0件を返した場合のハンドリング漏れ。loading / error / data（0件）/ data（1件以上） の4ケースを必ず実装するルールが long_term.md に記録済み（Sprint 47 追記）のため、以後同じミスを防ぐ。
