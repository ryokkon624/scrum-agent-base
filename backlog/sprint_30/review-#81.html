Sprint 30 Review

#81 — ログイン済みユーザー情報をRiverpodで保持し、
各画面の冗長なAPI呼び出しを解消する

Sprint 30 #81 refactor mobile

なぜこの改修が必要だったか

問題

ログイン時にバックエンドから返ってくる AuthUser（userId / email / displayName）を AuthState に保持していなかった。そのため、ホーム画面・My Tasks画面の初期化のたびに loadCurrentUserId() APIを個別に発行していた。画面を開くたびに本来不要なAPI呼び出しが発生し、表示が遅延する原因になっていた。

改修前
ログイン → tokenのみ保存、AuthUser破棄
AuthAuthenticated: ユーザー情報なし
ホーム表示 → loadCurrentUserId() API発行
My Tasks表示 → loadCurrentUserId() API発行
→
改修後
ログイン → token + AuthUser を state に保持
AuthAuthenticated: user フィールドを持つ
ホーム表示 → authState から参照のみ（API不要）
My Tasks表示 → authState から参照のみ（API不要）
画面初期化時の余分なAPI呼び出し（改修前）
2回
ホーム・My Tasks各1回 loadCurrentUserId()
画面初期化時の余分なAPI呼び出し（改修後）
0回
authState から参照するのでAPI不要
