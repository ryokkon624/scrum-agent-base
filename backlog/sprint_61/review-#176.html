Sprint 61 #176 refactor mobile No-op

なぜこの改修が必要だったか

問題

DTO やリクエスト/レスポンスモデルの配置が feature によって data/ 直下と data/models/ サブフォルダの2パターンに混在していた。開発者がファイルを探す際に「この feature はどちらのパターンか」を確認するコストが発生していた。

想定されていた不統一パターン
features/home/data/home_raw_data.dart
features/home/data/housework_dto.dart
features/shopping/data/shopping_attachment_dto.dart
models/ サブフォルダなしで直置き
→
統一後（期待パターン）
features/home/data/models/home_raw_data.dart
features/home/data/models/housework_dto.dart
features/shopping/data/models/shopping_attachment_dto.dart
全 feature が models/ サブフォルダに統一

実施内容（No-op）

調査結果

調査の結果、全 feature のモデルファイルはすでに data/models/ サブフォルダに整理済みであることを確認した。
Issue 作成時点では data/ 直下に散乱していたが、過去の実装（#177 のフォルダリネームを含む）の中で整理が完了していた。

調査した全 feature の現在の配置（確認済み）:

lib/features/auth/data/models/
auth_user.dart, login_response.dart, register_response.dart ✓
lib/features/home/data/models/
home_raw_data.dart, household_member_dto.dart, ... ✓
lib/features/shopping/data/models/
shopping_item_dto.dart, shopping_attachment_dto.dart, ... ✓
lib/features/my_tasks/data/models/
housework_task_dto.dart ✓
lib/features/housework_assign/data/models/
（housework_task_dto は my_tasks/data/models/ を参照）✓
lib/features/inquiry/data/models/
inquiry_dto.dart, ... ✓

AC別 実施内容

—
AC1
home/data/ 直下のモデルファイルが home/data/models/ に移動する

No-op。調査の結果、home/data/models/ には既に全モデルファイルが配置済みであることを確認。data/ 直下にモデルファイルは存在しなかった。

—
AC2
shopping/data/ 直下のモデルファイルが shopping/data/models/ に移動する

No-op。調査の結果、shopping/data/models/ には既に全モデルファイルが配置済みであることを確認。data/ 直下にモデルファイルは存在しなかった。

—
AC3
他 feature の data/ 配下も確認し、models/ サブフォルダなしで直置きされているモデルファイルがあれば移動する

No-op。全 feature を調査した結果、data/ 直下に直置きされているモデルファイルはなかった。auth / home / shopping / my_tasks / housework_assign / inquiry 全て data/models/ に整理済み。

—
AC4
移動に伴う import パスが全ファイルで更新される

No-op。ファイル移動が不要なため、import パスの更新も不要。
