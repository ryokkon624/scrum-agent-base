## PBI-008: GET /api/housework-tasks で8日以上前のタスクを取得しないようにする

**優先度**: 中
**ステータス**: 未着手
**作成日**: 2026-04-13

**As a** HwHubの開発者が
**I want to** タスク取得APIで不要な過去データを取得しないようにしたい
**So that** ホーム画面のパフォーマンスを向上させる

### Acceptance Criteria

- [ ] AC1: GET /api/housework-tasks のクエリに下限日付として「今日から7日前」を指定する
- [ ] AC2: 例）今日が4/13の場合、4/6以降のタスクのみ取得される
- [ ] AC3: 未来分のタスクは引き続き全件取得される（上限なし）
- [ ] AC4: 既存の他のAPIエンドポイントに影響を与えない

### 備考

- **優先順位の根拠**: HOME画面で表示しない範囲のデータを取得しない性能改善
- **依存関係**: なし
- **実装箇所**: バックエンド（Java/Spring Boot）のhousework-tasksエンドポイント
- **ブランチ**: 新規作成：feature/9-add-lower-limit-when-retrieve-tasks
- **コミット番号**: (ryokkon624/hw-hub-manage#9)

## PBI-007: ホーム画面「家事の未割り当て」カードの件数が0件で表示されることがある

**優先度**: 高
**ステータス**: 未着手
**作成日**: 2026-04-13

**As a** HwHubを使うユーザーが
**I want to** ログイン時や世帯切り替え時に家事の未割り当てカードの件数を正しく確認したい
**So that** 未割り当て家事の状況を正確に把握できる

### 事象

ログイン時や世帯切り替え時に「未割り当て」「今日～3日以内」の件数が0件で表示されることがある。常時0件ではなく、切り替え後に正しく表示されることもある。

### Acceptance Criteria

- [ ] AC1: ログイン後、未割り当てカードの「未割り当て」「今日～3日以内」の件数が正しく表示される
- [ ] AC2: 世帯切り替え後、未割り当てカードの件数が正しく表示される
- [ ] AC3: 既存の他のカード・機能に影響を与えない

### 備考

- **優先順位の根拠**: 正確な情報表示はUXの基本のため高優先
- **依存関係**: ①（PBI-006）と同ブランチで対応
- **調査観点**: データ取得処理のawait漏れ・非同期処理のタイミング問題が疑われる
- **ブランチ**: prefixは `fix/` とする（PBI-006と同ブランチ）
- **コミット番号**: (ryokkon624/hw-hub-manage#8)

## PBI-005: ホーム画面「おうちの様子」タスク件数の集計バグ修正

**優先度**: 高
**ステータス**: 完了
**作成日**: 2026-04-13

**As a** HwHubを使うユーザーが
**I want to** ホーム画面でその日のタスク総数を正しく確認したい
**So that** タスクを完了させても件数が変わらず、正確な状況を把握できる

### 事象

My Tasks画面でタスクを完了にすると、ホーム画面の「おうちの様子」カードで該当日のタスク件数が減る。

**再現手順：**

1. ホーム画面で「おうちの様子」カードの4/10のタスク件数が8件であることを確認
2. My Tasks画面で1件のタスクを完了にする
3. ホーム画面に遷移すると4/10のタスク件数が7件で表示される（期待値：8件）

### Acceptance Criteria

- [x] AC1: My Tasks画面でタスクを1件完了にした後、ホーム画面の「おうちの様子」カードで該当日のタスク件数が変わらない
- [x] AC2: 「おうちの様子」カードのタスク件数は完了・未完了を含む**総数**を表示する
- [x] AC3: 既存の他のカード・機能に影響を与えない

### 備考

- **優先順位の根拠**: 正確な情報表示はUXの基本のため高優先
- **依存関係**: なし
- **調査済み**: バックエンドAPIは問題なし。フロントエンドの集計ロジックが原因
- **実装箇所**: hw-hub-frontend のホーム画面コンポーネント（おうちの様子カード）
- **ブランチ**: 新規で作成。prefixは `fix/` とする
