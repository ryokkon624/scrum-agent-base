原因分析

🔍 HouseholdNameSection の原因

initState() で1回だけ householdNotifier から世帯名を取得して _controller.text にセットしていた。 世帯切り替えで householdNotifier の selectedHousehold が更新されても、 この State は再生成されないため古い値が残り続ける。

ref.listen を呼び出しておらず、世帯名の変化を一切検知できていなかった。

🔍 NicknameSection の原因

_controller = TextEditingController() と空で初期化しており、 ニックネームの初期値がそもそも表示されていなかった。

HouseholdSettingsState にニックネームのフィールドが存在しなかったため、 世帯切り替え後に members を再取得しても Section 側には反映されなかった。

改修前（どちらも）
initState で1回だけ初期値をセット
世帯切り替え時の変化を検知する仕組みなし
Notifier が members を再取得しても
controller が更新されない
→
改修後
HouseholdNameSection: ref.listen(householdNotifierProvider) で selectedHousehold 変化を検知 → controller 更新
NicknameSection: State に currentNickname フィールドを追加。Notifier が members 取得後に nickname ?? displayName を計算してセット
NicknameSection: ref.listen(householdSettingsNotifierProvider) で currentNickname 変化を検知 → controller 更新
