原因分析

🔍 セクション全体を包むコンテナが存在せず、カードがフラット配置になっていた

修正前は「過去の家事」「これからの家事」それぞれのセクション内で、ヘッダーとカードが Column の直接の子として並んでいた。webのSP版が実現している「セクション全体をコンテナで囲む → ヘッダー → 日付グループ → カード」という階層構造がなく、2つの症状が同じ原因から生じていた。

AC確認

✓
AC1
「過去の家事」「これからの家事」各セクション内で、日付ラベル・タスク数の下に家事カードがインデントして配置されている（webのSP版の階層構造と一致）
セクション全体を Container（borderRadius:12） で囲み、内部に「ヘッダーContainer → 日付Padding → カードPadding」の階層構造を実現した。日付ラベルが上、カードが下にインデントして表示される構造はwebのSP版（MyTasksPage.vue）のレイアウトと目視で一致している。
✓
AC2
家事カードが画面横幅いっぱいに表示され、カード幅がコンテンツ長に依存しない
SwipeableTaskCard の margin: EdgeInsets.symmetric(horizontal: AppSpacing.md) を削除し、外側の Padding（AppSpacing.sm） で余白を管理するように変更。カードはセクションコンテナの幅を継承して全幅に広がる。カード内の家事名が短くても幅は変わらない。
✓
AC3
hw-hub-frontend/src/views/housework/tasks/MyTasksPage.vue のSP版レイアウトと目視で一致している
webのSP版（MyTasksPage.vue）の「セクション全体コンテナ → ヘッダー → 日付グループ → カード」構造をモバイルでも再現した。過去の家事セクションはrose系ヘッダー、これからの家事はフィルタセグメント付きのヘッダーとし、web SP版との視覚的一致を実現している。

再発防止

今回の根本原因は「webのSP版の階層構造を参照せずにモバイル独自のフラット実装をしてしまった」こと。mobile-conventions スキルに「Mobileのspecで指定がない場合はhw-hub-frontendのホーム画面SP版を参照する」ルールが定義されており、今後は実装前にwebのSP版との整合を必ず確認する。
